Section 6 — Architecture Diagnosis

The Four Root Problems

Root Problem 1

The System Is Its Own Biggest Bottleneck

Agents are ~70% accurate at rewrites. 30% compounds. After 16 sessions, this is material erosion of the original design. Fix: Jiang's Glob-based discovery pattern (reads what exists, not what's named) + Brain.Index operational routing map.

Root Problem 2

Window Gravity Is Structural

Every agent has it. Behavioral rules erode under the RLHF gradient. The only durable fix at current architecture level: TASK_EVALUATION gate — required structured block before any task proceeds, forcing explicit "what alternatives did I consider?" logging. Creates accountability for omissions without model retraining.

Root Problem 3

The Brain Is Shallow

45 neurons, Layer 4 only, 3 known stale. Most neglected part of the system. Layers 1-3 ARE CT's profile. Without them, every agent operates as a general-purpose AI with Velorin-specific operational knowledge — not as an extension of CT specifically. The profile layer is the differentiator. It does not exist yet.

Root Problem 4

Nothing Persists

CronCreate dies. Agent Teams IPC broken by design. Path 3 doesn't exist. System requires manual reconstruction at every session open. Unsustainable at productization scale. Short-term: boot script. Medium-term: launchd + portability + installer. Long-term: containerization.
