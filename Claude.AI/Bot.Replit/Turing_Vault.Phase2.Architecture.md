1. CRITICAL ARCHITECTURE CORRECTION


The current adversarial paywall is wrong. It was built on a founder assumption — the founder is WAIS-IV 99.9th percentile verbal, competitive, ambition 100%. He would pay for the harder version. The median user arriving from a TikTok about AI replacing their job is not him. They are anxious and curious. They will take the free thing, feel something about it, and leave or share. They will not pay to feel the same thing harder.


Paywalling difficulty gates the exact population whose data needs to be collected. It kills the database to generate revenue from a purchase most users will not make.


2. NEW ARCHITECTURE


ALL DIFFICULTY LEVELS: FREE
Standard, Hard, Adversarial — all open. No gates. Database builds at full velocity with clean, complete, annotated records across all tiers. The rationale field — the highest-value data point in the product — gets filled in. This is the asset.


SINGLE PAYWALL: IQ VAULT — elevated to second hero on page
This is a genuinely different product. Not a harder version of the same test. A different experience: IQ scoring methodology, permanent ranked placement, leaderboard status. Value proposition is identity and status, not difficulty. Psychologically valid purchase for a different user segment — competitive, status-oriented users who want a score and a rank.


POST-RESULT BEER BUTTON: optional, results screen only
At the moment of emotional discovery — after the user finds out they could not distinguish AI from human writing. Not on the landing page. Never a gate.
Copy (exact): "I built this because I think this question matters. If it moved you, a coffee keeps the lights on. No pressure."
Single-click $2.71 button. Clearly marked skip.


3. WHY THIS MATTERS FOR THE DATABASE


If difficulty is paywalled:
- Majority of users hit a gate before completing a test
- Partial sessions = incomplete data = low-quality annotations
- Rationale field never gets filled
- Database fills with fragments
- Volume drops 60-80% vs fully open
- The long-term business asset is permanently compromised


If all difficulty is free:
- Full sessions, complete records, annotated rationale
- All difficulty tiers represented in the data
- Adversarial data has normalization context (user completed Standard first)
- Database builds at full velocity
