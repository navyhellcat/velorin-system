5. OAUTH AND CREDENTIALS

Project: core-craft-437617-m5
OAuth Client: Velorin Desktop (Desktop app type)

Redirect URI Fix
Desktop app OAuth clients default to http://localhost with no port. The server was configured to listen on port 3000. The credentials file had to be edited directly to change the redirect URI from http://localhost to http://localhost:3000.
This is valid per RFC 8252 — installed/desktop app OAuth clients may use any available loopback port. Google does not enforce port matching for Desktop type clients.
Command used to fix:
sed -i '' 's|"http://localhost"|"http://localhost:3000"|' gcp-oauth.keys.json
Token File
After running node auth.js and completing the browser authorization flow, tokens are saved to:
/Users/lbhunt/Desktop/Velorin/Velorin Code/velorin-gdrive-mcp/tokens.json
Tokens auto-refresh. The server writes updated tokens to disk on every refresh event. Token expiry should not cause server failure as long as the refresh token remains valid.
If auth ever breaks: run node auth.js again from the project folder. This regenerates tokens.json.

6. BUG FIXES AND DECISIONS — VERSION HISTORY
v1.3.0 — First complete build (Alexander approved)
* responseType: text fix on drive.files.export
* OAuth2Client constructed with clientId + clientSecret — fixes auto-refresh
* Token refresh listener writing to disk immediately
* Two separate shared drive constants — SHARED_DRIVE_FOLDER vs SHARED_DRIVE_SEARCH
* withRetry() for 429/503
* validateId() and escapeQuery() on all inputs
* try/catch around server.connect(transport)
* gdrive_overwrite_doc, gdrive_move_file, gdrive_trash_file added
v1.3.1 — Patch after Trey code review (4 confirmed bugs fixed)
* extractError() — 401 check now fires before response.data check. Re-auth hint was dead code in 1.3.0.
* Resource key — all five file-operating tools now accept resource_key. Resolved from caller or metadata. Passed on all subsequent calls for that file.
* Retry coverage expanded — now covers 403 rateLimitExceeded, 403 userRateLimitExceeded, 500, 502, 503, 504. Prior version missed the most common real-world quota failure mode.
* gdrive_overwrite_doc — inspects body structure before delete. Refuses docs with tables, TOC, or positioned objects. Honest about the limitation.
* gdrive_create_doc — trashes created doc on any post-creation failure. No more silent orphans.
* gdrive_move_file — refuses folders explicitly. Detects and logs cross-drive moves.
* Auth loopback — 127.0.0.1 and [::1] added alongside localhost in both auth.js and server.js.
