Let $V$ be the verifier agent, mapping $y$ to $z \sim P\_V(\cdot \mid y)$ over vocabulary $\mathcal{V}$.

Let $F$ be a Finite State Machine grammar defining exactly the language $\mathcal{L}\_{IES}$.

Constrained decoding restricts $P\_V$ at step $t$ based on the valid vocabulary subset $\text{valid}(F, z\_{<t}) \subseteq \mathcal{V}$:

$$P_V^F(z_t \mid z_{<t}, y) = \frac{P_V(z_t \mid z_{<t}, y) \cdot \mathbf{1}[z_t \in \text{valid}(F, z_{<t})]}{\sum_{v \in \mathcal{V}} P_V(v \mid z_{<t}, y) \cdot \mathbf{1}[v \in \text{valid}(F, z_{<t})]}$$

### 5\. PROOF 1 — DETERMINISM OF FSM-CONSTRAINED DECODING

Theorem: If the FSM grammar $F$ is well-formed over $\mathcal{V}$, FSM-constrained decoding guarantees $z \in \mathcal{L}\_{IES}$ with probability 1, strictly subject to the maximum sequence length bound $L\_{max}$.

Proof:

By definition, a string belongs to $\mathcal{L}\_{IES}$ if and only if it represents a path from the start state of $F$ to an accepting state of $F$.

At any decoding step $t$, the indicator function $\mathbf{1}[v \in \text{valid}(F, z\_{<t})]$ acts as a Dirac filter. It strictly zeros out the probability of any token $v$ that violates the state transitions of $F$.

Does the stochastic nature of the base model $P\_V$ introduce failure?

Because standard autoregressive Softmax outputs assign strictly non-zero probability to all tokens in $\mathcal{V}$, the denominator:

$$\sum_{v \in \mathcal{V}} P_V(v \mid z_{<t}, y) \cdot \mathbf{1}[v \in \text{valid}(F, z_{<t})] > 0$$

is guaranteed to be strictly positive as long as $\text{valid}(F, z\_{<t}) \neq \emptyset$. The sampling operation will always select a token permitted by the grammar. The base model's stochastic preferences merely dictate which valid path is chosen.

Failure Modes:

Determinism fails under exactly two edge conditions:

  1. Vocabulary Deadlock: The FSM transitions into a state where no single token in the LLM's Byte-Pair Encoding (BPE) vocabulary can advance the machine. The valid set is empty, the denominator is zero, and generation deadlocks. This is resolved by pre-compiling $F$ as a Deterministic Finite Automaton (DFA) directly over the token graph of $\mathcal{V}$.
  2. Context Truncation: The LLM emits tokens to satisfy the grammar but reaches the maximum context limit $L\_{max}$ before $F$ reaches an accepting state.


Provided the grammar is compiled correctly to avoid BPE deadlocks, and $L\_{max}$ strictly exceeds the valid path length, the probability of format compliance is mathematically absolute:

$$P(z \in \mathcal{L}_{IES}) = 1$$

Determinism holds. $\blacksquare$

### 6\. PROOF 2 — SELF-SIMILARITY AND THE RULE ACTIVATION GAP

Theorem: Delegating classification to a message header tag and mapping to FSM-constrained decoding eliminates the Rule Activation Gap. The gap is not relocated to the verifier; it is algebraically shattered.

Proof:

Let $C$ be the classification event (the payload requires IES).

Let $M$ be the mapping event ($z \in \mathcal{L}\_{IES}$).

If the verifier evaluated both, $P(M, C \mid y) = P(M \mid y, C) \cdot P(C \mid y)$. Because $y$ is a long reasoning trace, $P(C \mid y)$ would be subjected to attention decay. The Rule Activation Gap would merely be relocated to the verifier's classification step.

In Velorin's ATV implementation, the generating agent $G$ emits $C\_{header}$ (the tag content\\_type: analytical\\_reasoning) in the message header. This tag is generated before the long analytical payload, minimizing the attention distance to the system prompt. Thus, $G$ does not suffer the Rule Activation Gap when generating the tag.

The verifier reads $C\_{header}$ via standard transport middleware. This is a purely deterministic software routing step:

$$P(C \mid C_{header}) = 1$$

Once routed, the verifier executes the mapping. From Proof 1, we know:

$$P(M \mid y, C_{header}) = P_V^F(z \in \mathcal{L}_{IES}) = 1$$

The composed system $G \circ V^F$ achieves format compliance completely free of stochastic classification.

Residual Error Characterization ($\epsilon^+$):

The only residual stochastic failure is the producer hallucinating the tag on non-analytical content (False Positive, $\epsilon^+$). What happens if $y \notin \mathcal{L}\_{IES, content}$ but the tag is present?

Because $P(M \mid C\_{header}) = 1$, the verifier is forced to run FSM decoding on irrelevant content. The FSM constraints will violently coerce the base model to output the required syntax (e.g., <consensus\\_extraction>). The LLM will attempt to probabilistically fit the irrelevant input $y$ to this structural skeleton, resulting in output that is structurally perfect ($z \in \mathcal{L}\_{IES}$) but semantically vacuous.

Thus, $\epsilon^+$ equals the producer's tag-hallucination rate, but the format compliance of the system remains absolute. $\blacksquare$

### 7\. PROOF 3 — LATENCY COST ANALYSIS (THEORETICAL BOUNDS)

Theorem: FSM-constrained decoding operates in $O(1)$ extra time per token state transition. It reduces theoretical latency by bypassing the neural forward pass for fixed structural strings, bounding the latency cost linearly in $O(L\_F)$.

Proof:

Let $T\_V$ be the total verification time. Let $L\_F$ be the formatted sequence length. Let $S$ be the token generation speed of the local verifier (tokens/second).

  1. Theoretical Lower Bound:  
The unconstrained time to generate $L\_F$ tokens is $L\_F / S$.
  2. FSM vs. Unconstrained Decoding Speed:  
Does masking speed up or slow down the model? The LLM must still compute the dense matrix multiplications ($O(d^2)$) for all vocabulary logits at each step. Applying the FSM constraint naively would cost $O(|F|)$ per token. However, compiling $F$ into an indexed DFA makes the state transition and mask retrieval an $O(1)$ lookup. Applying the mask is $O(|\mathcal{V}|)$, a CPU operation that is negligible compared to the GPU forward pass.
  3. The Fast-Forward Acceleration:  
Crucially, the IES grammar requires fixed structural tokens (e.g., <velorin\\_divergence\\_evaluation>). When the DFA reaches a state where exactly one token string is valid ($|\text{valid}(F, z\_{<t})| = 1$), the inference engine fast-forwards. It appends the required token(s) immediately without executing the LLM's $O(d^2)$ forward pass.


If $\Delta$ is the total length of the fixed IES skeleton tokens, the effective number of neural evaluations drops to $(L\_F - \Delta)$.

$$T_V \approx \frac{L_F - \Delta}{S}$$

The latency cost scales strictly linearly $O(L\_F)$. FSM constraints objectively reduce the total neural compute. $\blacksquare$

### 8\. PROOF 3 — LATENCY COST ANALYSIS (OPERATIONAL THRESHOLDS)

Theorem: The ATV introduces a hard M/D/1 queueing bottleneck that becomes operationally destabilizing if message arrival frequency exceeds the inverse service time of the local silicon.

Proof:

We model the ATV MCP endpoint as a single-server queue. Let $\lambda$ be the arrival rate of analytical messages across the swarm (messages/second).

Assume a Mac Studio M4 Max with MPS backend running Qwen 3.6 (e.g., 14B or 32B quantized). Memory bandwidth dictates a generation speed of $S \approx 60$ tokens/second.

Assume an average IES-formatted output length of $L\_F = 600$ tokens, with $\Delta = 60$ fixed structural tokens.

$$T_V \approx \frac{600 - 60}{60} = \frac{540}{60} = 9 \text{ seconds}$$

The system enters infinite unbounded delay (catastrophic queueing failure) when the utilization factor $\rho \ge 1$:

$$\rho = \lambda \cdot T_V \ge 1 \implies \lambda \ge \frac{1}{T_V}$$

The Threshold:

$$\lambda_{critical} \approx \frac{1}{9} \approx 0.11 \text{ messages/second}$$

If the agent swarm generates more than 1 analytical message every 9 seconds, the ATV introduces compounding, observable delay into the communication loop. Below this threshold, it acts as a stable, fixed $\sim 9$-second transport-layer penalty. $\blacksquare$
