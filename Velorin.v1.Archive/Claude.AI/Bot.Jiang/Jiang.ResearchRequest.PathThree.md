BACKGROUND
Velorin's end state requires one Claude instance to programmatically call other Claude instances — Jiang, Trey, Todd, and future agents — without human browser navigation between sessions. This is Path 3: API-based multi-agent orchestration. It is the architectural requirement that enables fully automated end-of-day protocol, autonomous agent coordination, and Velorin operating as a real system rather than a collection of manually opened windows.


Path 3 is documented as the target in Velorin.Company.DNA and Velorin.Company.Operating.Standards. It does not yet have a concrete build roadmap.


RESEARCH MANDATE
Map the existing multi-agent orchestration landscape as of March 2026. The question is not theoretical — it is: what exists today that Velorin can use or build on to reach Path 3, and how fast?


SPECIFIC QUESTIONS TO ANSWER


1. Anthropic API multi-agent capabilities

3. Anthropic's own agentic products
Claude Code, Claude Desktop MCP, and any agent orchestration features Anthropic has shipped or announced. What is already in Anthropic's ecosystem that Velorin should be using?

5. Risks and failure modes
What are the known failure modes of multi-agent orchestration at this stage of the technology? What should Velorin build around vs. wait for?
