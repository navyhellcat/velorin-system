The Brain Activation Concept

Input decomposes into parts → parts trigger brain regions → program retrieves the right neurons → sub-agent reasons over the compiled payload → main agent gets complete context → one pass, no follow-on. A system that takes 2 minutes to produce a complete output beats 30 seconds requiring 5 follow-on rounds. Total time from problem to true solution is the metric.

Jiang2 modification to the brainstorm: The brain traversal should not be a pre-pass before the agent starts reasoning. It should be a callable MCP tool the agent invokes mid-reasoning as needed.
Why pre-pass fails
Mandatory Pre-Pass Problem
A mandatory pre-pass requires predicting which regions are relevant before reasoning begins. This works when the query maps cleanly to known regions.

It fails when reasoning mid-stream reveals a connection the decomposition didn't anticipate — the agent has no way to go back and get more context. The regions it needed were never activated.
Why callable tool wins
brain_retrieve(query_fragment)
Agent calls it exactly when it needs context, with exactly the fragment it needs at that moment.

For well-covered brain regions: one call, complete context. For sparse regions: multiple calls as reasoning develops. Over time as the brain fills, the multiple-call pattern disappears.

This architecture gracefully handles today's sparse brain vs. tomorrow's rich brain. The tool interface doesn't change. The brain content does.
Confidence
83%
Sub-Agent Crossover Point
When Regional Sub-Agents Become Correct
Sub-agent spawning per region makes sense when: (1) Brain has 500+ well-populated neurons across at least 5 distinct regions. (2) At least 3 regions are activated by a typical query. (3) Reasoning required per region is non-trivial (multi-step, extended thinking).

At that scale, parallel regional sub-agents — each with focused context, each producing high-quality regional answers — reduces total reasoning time substantially. Below that: single agent with callable brain tool.

Current state: Single agent with callable brain tool. Design for regional sub-agents, implement as single agent now.

Missing 1 — Original
Activation ≠ Retrieval. The Wikilink Mechanism Only Solves Half.
Retrieval (what the parser solves): given known entry neuron IDs, traverse the graph and compile a knowledge packet. Deterministic, fast, program-level. The YAML parser nails this.

Activation (what the parser doesn't solve): given an arbitrary natural language query, identify which neuron IDs are the correct entry points. This is a semantic similarity problem. Keyword decomposition in Phase 2 approximates it. But for novel queries that don't map cleanly to known region names, the parser has no entry point and returns nothing.

The hippocampal indexing model (Teyler & DiScenna, 92% confidence) stores pointers to distributed cortical patterns. The Brain's pointer graph IS this structure. What it's missing is the semantic embedding that lets the hippocampus receive a novel pattern and activate the right index entries.

Solution: add a vector layer as the entry-point discovery mechanism above ~200 neurons. Embed neuron titles/first sentences. Query against those embeddings to find entry points. Then use the pointer graph for traversal. This is the exact Layer 2 architecture Trey and Jiang both described.
Missing 2 — Original
The Output Contract Problem is Equal in Importance to Retrieval
The brain activation system solves the INPUT side. The output side has a separate structural problem that retrieval alone cannot fix (FollowBench ceiling, ~3 constraints).

The fix is a structured output contract block — the BOOT_PENDING fail-closed state machine pattern applied to output. The agent fills each field before delivering. Unfilled fields = response is incomplete = agent continues. No human follow-on needed.

The output contract + brain retrieval combination is what makes the single-pass metric actually achievable. Without the output contract, perfect retrieval still produces dropped fields.
OUTPUT_CONTRACT:
  box_relevance_assessed: [which boxes touched]
  cross_domain_implications: [each box → what changes]
  known_patterns_applied: [which CT patterns active]
  decision_options: [options, tradeoffs, historical risks]
  uncertainty_labeled: [what is unknown, confidence levels]
  completion_check: COMPLETE | NEEDS_MORE_CONTEXT
If NEEDS_MORE_CONTEXT → agent calls brain_retrieve again with the specific gap. Handles its own gap-filling. No human in the loop.
